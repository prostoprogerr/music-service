## Техническое задание на проект: Сервис подбора похожих музыкальных треков для диджеев ##

**1. Цель проекта**
Создание веб-сервиса, который позволяет пользователям находить музыкальные треки, похожие на заданный, на основе данных из открытого API Deezer. Сервис помогает диджеям, музыкальным редакторам и просто любителям музыки быстро подбирать композиции для плейлистов, анализировать жанровое сходство и сохранять результаты в личных плейлистах.

**2. Роли пользователей**
  - Гость (неавторизованный пользователь) - может искать похожие треки, просматривать результаты, создавать плейлисты (с привязкой к сессии).  
  - Авторизованный пользователь - имеет те же возможности, но плейлисты привязываются к его учётной записи, доступны с любого устройства.  
  - Администратор - управляет моделями через админку Django (добавление/удаление треков, просмотр статистики).

**3. Модели данных**
  - Track (Композиция) - хранит информацию о музыкальном треке: название, исполнитель, ссылка на превью (preview_url), идентификатор Deezer.
  - Similarity (Сходство) - связывает два трека (исходный и похожий) с коэффициентом схожести (score). Используется для аналитики и истории.
  - Playlist (Плейлист) - содержит название, владельца (пользователь или сессия) и список треков через ManyToMany.

**4. Ключевой функционал**
  - Поиск похожих треков по исполнителю и названию с использованием Deezer API.
  - Отображение найденных треков с возможностью прослушивания 30-секундных фрагментов.
  - Создание и сохранение плейлистов из выбранных треков.
  - Просмотр списка своих плейлистов и их содержимого.
  - Регистрация и авторизация пользователей (логин/пароль).

**5. Внешние интеграции и аналитика**
  - Deezer API - получение информации о треках и похожих исполнителях, ссылок на аудио-превью.
  - Внутренняя аналитика - на основе сохранённых связей Similarity система вычисляет и отображает рейтинг схожести (match_score), который может использоваться для построения простых графиков (например, распределения популярности жанров).
